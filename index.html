Manifesto

core beliefs and purpose of KaratsubaLabs

1. Promote the creation and use of free and open source software

KaratsubaLabs seeks to self host as many open source services as possible. This includes a collaborative document editor (replacement for google docs), kanban task manager (replacement for trello etc), a gitea instance (replacement for github), and many more. The goal is to allow a user to move off of (or at least no longer be dependent on) proprietary platforms.

2. Learn and research software by creating things yourself

KaratsubaLabs is also a faucet for learning. Most of the homebrew software was written out of a combination of necessity and desire to learn something new. KaratsubaLabs gives you an 'excuse' to write software, and also gives you an 'excuse' to scale your code.

A goal on top of the goal to create a free and open source platform is to create a free and open source platform that you wrote yourself. Another way to think about it is that the open source software we are currently using are simply placeholders before you write the software yourself.

3. Fight back against bloat and aim for software minimalism

Finally, KaratsubaLabs aims to fight back against modern software bloat. Software that is written aim to provide simpler and less heavy alternatives against existing options. This goal is in line with the unix philosophy of 'doing one thing and doing it well'. In addition, configurability and user control is also heavily emphasized. Code that is written should be easy to hack on.
